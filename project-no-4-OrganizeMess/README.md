# 🗂️ Organize Mess

A safe, dry-run-first Streamlit application for analyzing a messy folder, backing it up,
building a full execution plan, waiting for explicit human approval, and only then
organizing files — as **copies**, never by moving or deleting originals.

Built for **Task 4 – Organize the Mess (The Files You Forgot)**.

---

## Project overview

Most "auto-organize my folder" tools are risky: they move or delete files immediately,
with no preview and no way to undo. This project takes the opposite approach:

1. **Generate** a realistic, disposable synthetic `Downloads/` folder (never your real one).
2. **Back it up** verbatim before touching anything.
3. **Scan** it and compute real statistics (types, duplicates, empty files, large files, old files).
4. **Explain** in plain English what "clean" means for this run.
5. **Build a full dry-run plan** — every move, rename, flag, and recommended deletion — without
   executing a single operation.
6. **Wait for explicit approval** via a checkbox before the Execute button is even enabled.
7. **Execute** by *copying* organized files into a new folder — originals are never modified,
   moved, or deleted.
8. **Report** the results as CSV, Excel, JSON, and plain-text summaries, plus an execution log.

## Assignment mapping

| Assignment step | Where it lives |
|---|---|
| Messy folder to organize | `data/Downloads/` (synthetic, auto-generated) |
| Backup before touching anything | `modules/backup.py` → `data/Downloads_Backup/` |
| Scan & statistics | `modules/scanner.py` |
| Cleaning brief | "📋 Cleaning Brief" page in the app |
| Dry run / execution plan | `modules/planner.py` → "🧪 Dry Run" page |
| Approval gate | "✅ Execution" page (checkbox gates the Execute button) |
| Execute (copy-only) | `modules/organizer.py` → `generated/Organized/` |
| Verification / reports | `modules/reporter.py` → "📊 Reports" page |

**Note on the two intentional deviations from a literal reading of the brief, and why:**

- **Dummy data instead of a real folder.** The source folder is synthetic
  (`data/Downloads/`, auto-generated by `modules/dummy_generator.py`) rather than a real
  personal folder, so there is nothing real to lose while testing. This is a substitution
  for safety/demo purposes only — the reusability section below explains how to point the
  tool at a real folder later.
- **Backup is kept, not skipped.** Because the source data is synthetic, an earlier draft
  of this brief considered skipping the backup step entirely. It was kept anyway: it costs
  nothing here (the data is disposable) and it demonstrates the core safety habit the whole
  assignment is built around — you should always have a backup before an automated tool
  touches your files, real or fake.

## Features

- 🎲 One-click generation of a realistic messy folder (~100–150 files, nested folders,
  multiple duplicate patterns, empty files, oversized files)
- 🛡️ Mandatory backup step with on-screen confirmation (file count + total size)
- 🔍 Rich folder scan: type/category breakdown, largest/smallest files, duplicate names,
  duplicate content (SHA-256), empty files, old files, recently modified files
- 📋 Plain-English cleaning brief, tailored to your latest scan
- 🧪 Full dry-run plan broken into MOVE / RENAME / FLAG / recommended-DELETE sections,
  each clearly labeled and never executed automatically
- ✅ Explicit approval checkbox gating the Execute button
- 🚚 Copy-only execution into categorized folders (Images, Documents, Videos, Audio,
  Archives, Programming, Data, Others), with collision-safe automatic renaming
- 🗑️ Recommended-deletion items copied into a separate review folder instead of being deleted
- 📊 Reports in CSV, Excel (multi-sheet), JSON, and plain text, plus a persistent execution log
- 🎨 Modern dark dashboard UI: metric cards, progress bars, status badges, search & filters,
  expandable sections, charts

## Safety mechanism

1. **Backup first, always.** `modules/backup.py` copies `data/Downloads/` verbatim to
   `data/Downloads_Backup/` before any scan or plan runs, and the UI confirms the file
   count and total size copied.
2. **Dry run before anything happens.** `modules/planner.py` computes the *entire* plan
   (moves, collision-safe renames, duplicate/large/old/empty flags, and recommended
   deletions) purely from scan data — no filesystem writes occur during planning.
3. **Explicit approval gate.** The Execute button on the "✅ Execution" page is disabled
   until the "I have reviewed the plan" checkbox is ticked.
4. **Copy-only execution.** `modules/organizer.py` uses `shutil.copy2` exclusively.
   Originals in `data/Downloads/` and `data/Downloads_Backup/` are never opened for
   writing, moving, or deletion.
5. **No automatic deletion, ever.** Files identified as redundant duplicates are copied
   into `generated/Organized/_Recommended_For_Deletion/` for the human to inspect and
   delete manually if they agree. The app itself never calls a delete operation on
   anything in `data/`.

## Dummy dataset explanation

`modules/dummy_generator.py` builds a synthetic `data/Downloads/` folder (wiped and
regenerated on each run, seeded for reproducibility) containing:

- ~100–150 files across PDF, Word/Excel/PowerPoint stand-ins, ZIP, TXT, CSV, JSON, HTML,
  Python, images, video, and audio
- Nested folders (`Old Stuff/2023 backup`, `Work Files`, `Screenshots`, `Random`)
- **Exact duplicate files** — same name, same content, different folder
- **Content duplicates** — different filename, identical bytes (`*_copy` pattern)
- **Name-only collisions** — same filename, different content, different folders
- Empty (0-byte) files
- A handful of oversized files (6–14 MB, above the 5 MB "large file" threshold) — kept
  intentionally small so the whole demo project stays lightweight to download and run
- Backdated modification times so "old" and "recently modified" filters have real data
  to work against

## Reusability — pointing this at a different folder later

Every path in this project is derived from a single constant in `modules/utils.py`:

```python
SOURCE_DIR: Final[Path] = BASE_DIR / "data" / "Downloads"
```

To reuse this tool on a real folder later:

1. Open `modules/utils.py`.
2. Change `SOURCE_DIR` to point at the folder you actually want to organize (an absolute
   `Path(...)` works fine).
3. Skip the "Generate dummy Downloads folder" button on the Dashboard — go straight to
   "Run backup".
4. Everything downstream (backup location, scan, plan, organized output, reports) adapts
   automatically — no other code changes are required.
